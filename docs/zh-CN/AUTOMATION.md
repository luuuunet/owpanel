# OWPanel 自动化功能指南（小白版）

> 本文用**大白话**说明「自动化」菜单里每个功能是干什么的、和宝塔 / 1Panel / 阿里云 / AWS / Google Cloud 有什么对应关系，以及怎么一步步配好。

---

## 1. 自动化到底解决什么问题？

买一台服务器后，你通常要反复做这些事：

| 日常烦恼 | 自动化能帮你 |
|---------|-------------|
| 网站半夜挂了，第二天才发现 | **服务监控 + 自动重启**、**外网可用性探测** |
| 忘记备份，硬盘坏了数据全没 | **定时备份**（类似云厂商「快照」） |
| SSL 证书过期，浏览器报红 | **SSL 自动续期** |
| CPU/内存满了不知道 | **资源告警 + Webhook 推送到手机** |
| 总要登录 SSH 跑脚本 | **计划任务 Cron** |

OWPanel 把上述能力放在左侧 **「自动化」** 菜单里，**单机 All-in-One**，不用分别买云监控、云备份等多个产品。

## 3. 推荐新手：一键建站保护

路径：**自动化 → 自动化运维 → 入门指南**

点击 **「一键开启建站保护」**，面板会自动：

1. 开启自动运维、SSL 自动续期、网站健康检测  
2. 为 Nginx / MySQL / PHP 等已安装服务开启 **监控 + 自动重启**  
3. 为所有运行中的网站创建 **外网可用性探测**（每 5 分钟）  
4. 为所有网站创建 **每日 2:00 备份任务**  
5. 为所有数据库创建 **每日 3:00 备份任务**  

> 类似把 AWS Lightsail 的「自动快照 + 告警」和宝塔的「守护 + 计划备份」合成一次点击。

之后建议在 **策略设置** 里填写 **Webhook**，才能收到钉钉/飞书/企业微信通知。

## 4. 各功能详细说明

### 4.1 自动化运维（总控台）

**是什么：** 整个自动化的「指挥中心」，看资源用量、服务状态、快捷跳转到其他模块。

**子功能：**

| 标签页 | 作用 |
|--------|------|
| 入门指南 | 对比表、学习路径、一键预设 |
| 运维概览 | CPU/内存/磁盘仪表盘 |
| 监控列表 | 勾选软件的「启用监控」「自动重启」 |
| 运维日志 | 自动重启、告警、SSL 续期等记录 |
| 策略设置 | 检测间隔、Webhook、资源阈值、内存释放 |
| 网站检测 | 健康分、SEO/安全头/死链建议 |

**和云厂商区别：** 阿里云/AWS/GCP 的监控主要看**机器指标**；OWPanel 还能直接对 **Nginx/MySQL 进程** 做保活重启。

---

### 4.2 可用性监控

**是什么：** 像用户一样定期访问你的网址，测能不能打开、响应多快。

**快捷操作：**

- **从网站一键导入** — 不用手填 URL  
- 探测间隔：1 分钟（核心站）/ 5 分钟（推荐）/ 15 分钟（内网）

**对应云产品：** Google Cloud「Uptime Check」、AWS Lightsail 自定义指标告警。

---

### 4.3 备份

**是什么：** 定时把网站文件或数据库打包存起来，类似「快照」。

**快捷模板：**

- **备份全部网站** — 每天 2:00  
- **备份全部数据库** — 每天 3:00  

还可配置 FTP / SFTP / WebDAV / OSS 远程存储。

**对应云产品：**

- AWS Lightsail：**Automatic snapshots**（每日自动快照）  
- 阿里云轻量：默认需**手动快照**，可用 Cron + API 半自动  
- GCP：**Snapshot schedules**（磁盘快照计划）

---

### 4.4 计划任务（Cron）

**是什么：** 定时执行 Shell 脚本，如清理日志、续期证书、释放内存。

**内置模板：** Docker 清理、Nginx 重载、MySQL 优化、面板备份等（打开页面即可看到）。

**对应云产品：** 阿里云「命令助手」+ 自己写定时；AWS/GCP 多用 EventBridge / Cloud Scheduler + 脚本。

---

### 4.5 其他自动化相关菜单

| 菜单 | 一句话说明 |
|------|-----------|
| A/B 测试 | 产品分析、实验分流（云厂商一般无内置） |
| DevOps | CI/CD 部署、CVE 扫描（管理员） |
| K8s 集群 | 轻量 K3s 或接入已有集群（管理员） |
| 服务器集群 | 多机负载均衡、MySQL 主从 |
| 日志管理 | 日志保留天数、自动清理 |
| 扩展 | 事件钩子脚本 |
| 缓存&安全 | CDN、防火墙、WAF |

## 5. 名词解释

| 名词 | 解释 |
|------|------|
| **Webhook** | 一个 https 地址，出问题时面板往这里 POST 消息，钉钉/飞书机器人会转发到群 |
| **Cron** | Linux 定时任务语法，如 `0 2 * * *` = 每天 2 点 |
| **快照 / 备份** | 某一时刻数据的完整拷贝，用于恢复 |
| **可用性探测** | 模拟访客访问 URL，比「进程在跑」更接近真实体验 |
| **服务保活** | 发现 Nginx/MySQL 停了自动 `restart` |
| **K3s** | 轻量 Kubernetes，单机或小集群够用 |

## 6. 从云厂商迁到 OWPanel 的建议步骤

1. **先在入门指南点「建站保护」或「运维告警」** — 快速对齐基础能力  
2. **把云控制台里的告警邮箱，改成 Webhook** — 统一推送到 IM  
3. **备份：** 若以前用 AWS 自动快照，在 OWPanel 备份页点「备份全部网站/数据库」  
4. **监控：** 若以前看 GCP Uptime Check，在可用性监控点「从网站一键导入」  
5. **脚本：** 把云助手/用户数据脚本迁到 **计划任务**，用 Cron 表达式定时

## 7. OWPanel 独有、云厂商面板较少提供的

- **网站健康检测**（HTTP、SSL、安全响应头、SEO、死链）

- **内置 DevOps / CVE 扫描**

## 8. 常见问题

**Q：一键建站保护会覆盖已有备份吗？**  
A：不会。已存在同网站/同数据库的备份任务会跳过。

**Q：Webhook 怎么填？**  
A：在钉钉/飞书/企业微信创建群机器人，复制 Webhook URL 粘贴到「策略设置」。

**Q：和云监控比缺什么？**  
A：OWPanel 不提供全球多地域探测点、SMS 短信通道需自建 Webhook；多机大规模集群建议配合云原生监控（Prometheus 等）。

**Q：Cron 和备份有什么区别？**  
A：备份页是「打包网站/数据库」专用；Cron 可跑任意命令。备份底层也靠计划调度执行。
